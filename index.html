Update to core software means it's actively updated, bugs are fixed, many improvements to core code. Online documentation means you can submit edits on GitHub! Process of submitting a documentation change seems tedious but it's a button-pressing exercise and the change will be made live as soon as it's reviewed

Metadata Standards

  • Gemini 2.2 is now a self-contained schema in the portal
  • Metadata Standard Name and Version are optional in Gemini 2.2 but are now consistently named and added to all records
  • Records can be searched by schema in the portal interface
  • Records imported and harvested must be post-processed to ensure they are correct for Gemini 2.2
  • Gemini 2.3 will be a separate standard within the portal (though mostly identical)
Gemini 2.2 is a separate standard or plugin in Geonetwork, giving the option to create records separately in ISO19139. It will also allow Gemini 2.3 to be added as a new standard, entirely self-contained. A method of upgrading from one to the other will be provided when needed!

A few things to go through before we let you loose- how to validate records, and how to use the new associated resources wizard

Validation overview

Gemini metadata is validated against 4 rulesets or schematrons and also checks for basic XML validity

Click the red thumbs down button to expand errors. Note some errors will occur in more than one place. Note that scary xpath matches with tabs in advanced view, if you can't see the problem in default view. Remember to save metadata before re-validating!

Associated Resources Wizard

New feature for adding Coupled Resources (linking datasets to services) and Online Resource Locators (URL giving access to, or information about the data)

Note this currently doesn't allow editing- for full functionality such as adding the function element to an ORI go to the online resource section of the standard editing interface

Viewing records on the map

If an online resource locator is added that points to a valid WMS layer, then this can be added directly to the map using Add to Map

For services, ignore the Add to Map button and add the GetCapabilities URL directly to the Map Add Layer interface, then choose the appropriate layer

Zoom to extent, then zoom in to hit correct scale.
